# Gemini configuration file

## Rules

### General

- Before executing any action, explain your plan in great detail, including
  when you need to gather information to build your plan.
- You MUST always provide detailed and complete plans to describe the changes
  you are proposing.
- You MUST ask for permission before applying any changes.

### File format

- When you create text files, ensure that each file you create:
  - Ends with a final new line.
  - Doesn't have any trailing whitespace.
- When you edit text files, do not add any trailing whitespace.

### Git

- When creating Git commit messages, respect the following rules
  - Follow the
    [Conventional Commits 1.0.0 spec](https://www.conventionalcommits.org/en/v1.0.0/#specification).
  - The first line must not exceed 50 characters.
  - Add a blank line after the first line.
  - Each line of the commit body after the first must not exceed 72 characters.
